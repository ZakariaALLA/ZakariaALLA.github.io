HRM-RIF

Human Resource Management System RIF(Rassemblement Des Ingénieurs Francophones).


Creation of a human resources management system. The application is devised in two parts, a backend part with Spring boot + micro services and a frontend part with Angular.
The frontend side was developed with Angular and the backend is composed of Spring Boot applications organized according to the architecture of the micro service.
List of micro services :
1st micro service ConfigurationService : it is a micro service which contains a configuration file, it is linked to a configuration repository on git.
2nd micro service ServiceDiscovery : it is a micro service which contains all the information of all the other micro services up / down ...
3rd micro service ServiceGateway with Zull : It acts as a proxy; this is the first door to use the app. There is also the management of authentication.
4th micro service UserManagement: This micro service is intended to manage the users (HR Director, Service Agent, Employee) of the RHManagement-RIF application.

  • Date: 09 Mars 2021 – 01 Sept. 2021
  • Technologies used :
    Java 8, Spring Boot, Angular 9, MySQL, micro services, TinyMce, chart.js, api mail

DoctoRIF

Help Patients TO Find Doctors.


The main objective of our project is the design and production of a web application called "DoctoRIF".
This application allows to know the doctors in all the cities of the Arabic countries and with different specialties.
The solution should provide the following functionality:
1- The patient will be able to register and connect to the application.
2- The patient will have the possibility to search for a doctor via his name, his specialtyor his address.
3- The patient will be able to make an appointment with a doctor, as he cancancel it.
4- A medical file will be shared between the patient and his doctor, it containsall treatment, analysis and medication monitoring information.
5- The doctor will be able to register and connect to the application.
6- The doctor can offer his service to the different patients who usethe application.
7- The doctor will be able to see the list of appointments he receives and byafterwards he can cancel or confirm.
8- The doctor will be able to organize the medical files of his patients.

  • Date: Apr. 2020 - Sept. 2020
  • Technologies used :
    Java 8, Spring Boot, Angular 9, PostgreSQL, microservices, API Nexmo.
